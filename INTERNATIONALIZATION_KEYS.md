## Usage Instructions

To add support for a new language:

1. Create a new `values-{language_code}` folder in `res/` (e.g., `values-es` for Spanish, `values-fr` for French)
2. Create a `strings.xml` file in that folder
3. Copy all the keys from the main `strings.xml` file
4. Translate the values to the target language
5. Keep the same key names, only change the values

### Example for Spanish (`res/values-es/strings.xml`):
```xml
<resources>
    <string name="app_name">Park45 Medidor</string>
    <string name="settings">Configuraci√≥n</string>
    <string name="cancel">Cancelar</string>
    <!-- ... continue with all other strings -->
</resources>
```

### Important Notes:
- **Format Specifiers**: Some strings contain format specifiers (e.g., `%1$s`, `%2$s`) - these should be preserved in translations
- **Emoji Characters**: Emoji characters in strings like `log_viewer` should be kept as-is or replaced with culturally appropriate equivalents
- **Number Formatting**: The `%1$.1f%%` format in `update_progress` should be preserved exactly as it's used for number formatting
- **HTML Tags**: If any strings contain HTML tags, they should be preserved in translations
- **Context**: Consider the context when translating - some terms may have specific meanings in the parking/payment domain
